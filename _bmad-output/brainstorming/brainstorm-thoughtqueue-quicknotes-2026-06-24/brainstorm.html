01

Cursed Genie

wish → curse → patch

You rub the lamp and ask for effortless capture. The genie grants every wish exactly, then twists it. Each twist becomes the next design constraint.

🪔
Wish Grab text from anywhere with one move — right-click context action and a key command to turn a selection into a note. Plus quick-capture from the menu-bar dropdown.
Curse Granted — but now you have a pile of nameless fragments and nowhere they belong.
Patch Auto-generate a title for every note, no manual typing.
Wish Then never make me file anything either.
Curse Granted — titled notes, all dumped in one heap.
Patch Auto-organize notes into categories.
Wish Make the smarts private — I don't want a cloud reading my notes.
Curse Granted, but only if it stays offline and yours.
Patch A small local LLM powers auto-titling + auto-categorization. Private, offline.
Wish Don't interrupt me, but let me fix a wrong guess.
Curse Granted — a popup that nags, or one that vanishes too fast.
Patch Transient toast under the menu-bar icon showing the auto title + category with an Edit button; auto-dismisses on a configurable timeout.
Wish And make sure nothing ever collides or gets stranded.
Curse Granted — but the metadata could trap your notes inside the app. (This tension gets resolved later.)
Patch Each .md note carries YAML frontmatter (title, category, date); filename = slug + timestamp, so files never collide and stay self-contained & portable.
02

Ideation Relay Race

pass the baton, fast

No stopping to deliberate. Each idea hands off to the next runner. The first leg reframes the whole race.

⟲ REFRAME — the move that changed everything The app is a thin shell over the filesystem, not a note silo. Notes are plain .md files in a local folder structure, fully editable in any text editor outside the app. The filesystem is the source of truth; the app just creates, updates, organizes.
tension surfaced
This challenges the YAML idea. Maybe metadata should live in the structure itself — category = folder, title = filename — to keep the files clean for external editing.
dates
Date metadata comes from the filesystem (created/modified timestamps). No DB needed for dates — reinforces app-as-shell.
live sync
The app watches the store folder and live-reflects external changes (edits, moves, deletes from other tools). True two-way sync with the filesystem.
open-with
Destinations beyond Claude: Gemini and other AI tools, the Zed editor.
context handoff
"Open with" can inject an @-style reference to the note's full path into the target app (e.g. open Claude pre-loaded with @/path/note.md so the session can read the file). Not just launch — hand off context.
two action types
(1) Command — run a shell command with the path as input, e.g. zed {path}. (2) App + input — activate an app, then inject via simulation, either the @{path} reference or the full pasted document text (reuses the existing CGEvent keyboard-sim approach).
retrieval
Fast retrieval = fuzzy search across all note titles.
default click
A user setting controls default click behavior on a note: (a) run the default open command, (b) render markdown in-app, or (c) edit raw text in-app.

// GUARDRAIL · NORTH STAR
The line isn't a single feature — it's purpose. This stays a quick personal note app and never drifts into a knowledge-base, document-management, or corp tool. Scope creep = purpose creep.

04

Outlive the App

invented · durability

Imagine the app is gone and unmaintained in five years. What must be true of the folder so your notes stay fully usable?

⟨ here lies ThoughtQueue.app · 2026 – 20XX ⟩
the survival test: when the app dies, the only thing lost is convenience.
✕ acceptable to lose
  • Quick taskbar access
  • The toast popups
  • The local LLM auto-smarts
  • The search index
  • App configs
✓ never lost
  • The .md notes themselves
  • Categories (= folders)
  • Titles (= filenames)
  • Dates (= fs timestamps)
  • Full content & organization
// DECISION · radical data minimalism
The notes are the only thing worth persisting. Everything else is disposable helper functionality. The store folder should contain only notes — no app-specific clutter.
05

Job to Be Done

real usage moments

Forget features for a second. What are you actually hiring this app to do, in the moment, in real life?

"In a meeting, capture todo-list notes for later reference, so I can look things up and get people answers."

Underlying job → grab actionable todos fast mid-meeting, then retrieve to follow up. Implies markdown checkboxes / todo items + easy retrieval of action items.

"When I see text I want to reference later or build into a spec/plan/document, capture it, so I can later find it and assemble it into a document."

Underlying job → collect fragments over time and accrete them into a larger doc.
design response
Capture supports both modes: create a new standalone note, or append the captured text onto an existing note you're already building — serving the accretive "grow a doc from fragments" job without bespoke document-management features.
Append-target selection → a popup picker to choose which note to append to.
// THE WORKING DOCUMENT
A designated "working" doc as the default sink

You can quickly designate a current catch-all where all captures land by default unless a different target is specified. A powerful default that serves the accretive job with near-zero friction.

Synthesis · the payoff

It was one architecture all along.

connection 01
The working document closes the genie's curse

The default-sink "working doc" resolves the original Cursed-Genie tension. It delivers capture-without-thinking (zero decisions) and avoids the nameless-landfill problem, because captures accrete into a known target instead of spawning orphan notes.

connection 02
Three ideas were really one

App-as-shell, filesystem-as-database (folder / filename / timestamp), and "notes are the only durable thing" are a single coherent architecture: radical portability.

connection 03
The real breakthrough = context handoff

The breakthrough feature is open-with-as-context-handoff — the @path reference into Claude — not the basic launcher. It's what makes this more than a notes app while staying minimal.

connection 04
Killing frontmatter keeps it honest

Dropping YAML frontmatter to keep files clean for external editing is the move that makes the whole filesystem-shell philosophy internally consistent.
